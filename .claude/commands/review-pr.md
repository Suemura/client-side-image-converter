Github Pull Requestのコードレビューを行ってください。

PR: $ARGUMENTS

## 手順

※ 呼び出しプロンプトに変更概要（タスクの要約・変更ファイル一覧・実装意図）が含まれる場合はそれを起点にし、レビューは diff 範囲とその直接の参照元 / 参照先に集中する。diff 範囲外のコードベース探索はしない。

1. `gh pr view $ARGUMENTS` でPRの概要を確認
2. `gh pr diff $ARGUMENTS` でdiffを取得し、変更内容を把握
3. 以下の観点でレビューを実施:
   - コーディング規約・命名規則の遵守（CLAUDE.md / `.claude/rules/` 参照）
   - バグや潜在的な不具合
   - セキュリティ上の問題（XSS, インジェクション等）
   - プライバシーファースト原則の遵守（画像データがサーバーに送信されないこと）
   - パフォーマンスへの影響
   - 型安全性（TypeScriptの型定義）
   - エッジケースの考慮漏れ
   - `src/utils/` のロジック変更に対応するテストの有無
4. 問題がある場合、インラインコメントとしてPRに投稿

## インラインコメントの投稿

レビュー指摘がある場合、以下の形式でGitHub APIを使ってインラインコメントを投稿してください:

```bash
cat > /tmp/review_comments.json << 'EOF'
{
  "commit_id": "{最新のコミットSHA}",
  "event": "COMMENT",
  "body": "このレビューは Claude Code (AI) によるものです。\n\nサマリーテーブルをここに記載",
  "comments": [
    {
      "path": "対象ファイルパス",
      "line": 行番号,
      "body": "指摘内容"
    }
  ]
}
EOF

cat /tmp/review_comments.json | gh api repos/{OWNER}/{REPO}/pulls/{PR_NUMBER}/reviews --input -
```

## レビュー本文のヘッダー

レビューの `body`（サマリー）の冒頭には必ず以下の一文を入れてください:

> このレビューは **Claude Code (AI)** によるものです。

## コメントのフォーマット

各指摘には以下のプレフィックスを使い分けてください:

- **[重要]** 修正必須の問題（バグ、セキュリティ、型エラー等）
- **[改善]** 強く推奨する改善（可読性、保守性）
- **[軽微]** 軽微な改善提案（命名、フォーマット等）
- **[質問]** 意図の確認や質問

## 出力

最後にレビュー結果のサマリーを表示してください:
- 指摘数（重要 / 改善 / 軽微 / 質問）
- 全体的な所感
