# 完了前のドキュメント同期とレビュー（docs-sync / reviewer エージェント委譲）

## 原則
タスクを「完了」としてユーザーに提示する前に、**reviewer エージェントを起動して**独立したコンテキストでレビューを受ける。自分自身でレビューしてはならない。

## なぜサブエージェントか
- 同じコンテキストでの自己レビューはアンカリング・確認バイアスにより甘くなる
- 独立したコンテキストのレビュアーは客観的に評価できる
- メインのコンテキストウィンドウを消費しない

## 手順
1. 変更がドキュメント記載事項（コマンド・構成・ワークフロー・ユーザー向け機能）に触れる場合は、先に **docs-sync エージェント**を起動してドキュメントを同期する
2. **reviewer エージェント**を起動する
3. reviewer に変更の概要（何を実装したか、どのファイルを変更したか）を伝える
4. reviewer のレポート（Pass/Fail + 問題リスト）を確認する
5. **Fail が 1 つでもある場合**: 問題を修正し、再度 reviewer を起動する
6. **全項目 Pass**: ユーザーに提示してよい

## 適用タイミング
- コード変更を伴うタスクの完了時
- 複数ファイルにまたがる変更の完了時
- ※単一ファイルの軽微な修正（typo 等）は対象外（docs-sync も同様にスキップしてよい）
